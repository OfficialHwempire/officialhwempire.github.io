# OAuth 2.0 주요 컴포넌트 & Authorization Code Grant


## 1. OAuth 란?

OAuth는 사용자 비밀번호를 직접 공유하지 않고도,
외부 애플리케이션이 사용자의 리소스에 제한된 권한으로 접근할 수 있도록 하는 표준 인가(Authorization) 프로토콜입니다.
주로 소셜 로그인 및 외부 API 접근 제어에 사용됩니다.

대표적인 예로는 네이버, 구글, 카카오 로그인 연동 서비스가 있습니다.

## 2. OAuth 주요 컴포넌트

  - Resource Owner : 사용자
  - Client: OAuth를 통해 리소스 접근을 요청하는 애플리케이션
  - Authorization Server : 사용자 인증 및 권한 동의 후 토큰을 발급하는 서버
  - Resource Server: Access 토큰을 검증하고 실제 리소스를 제공하는 서버 

## 3. Authorization Code Grant 란?

Authorization Code Grant는 OAuth 2.0에서 가장 널리 사용되는 인증 방식으로,
인가 코드를 한 번 발급받은 후 이를 토큰으로 교환하는 2단계 인증 흐름입니다.
Access Token이 클라이언트에 직접 노출되지 않아 보안성이 높습니다.

## 4. Authorization Code Grant 흐름 


1. Client가 Authorization Server에 인가 요청을 보냄
2. 사용자가 로그인 후 권한 제공에 동의
3. Authorization Server가 Authorization Code를 Client에게 전달
4. Client가 Authorization Code로 Access Token 요청
5. Authorization Server가 Access Token(및 Refresh Token)을 발급
6. Client가 Access Token을 사용해 Resource Server에 접근
