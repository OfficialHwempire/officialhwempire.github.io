# Spring Cache 의 기능들

## 0. Spring Cache 어노테이션 

Spring Cache 어노테이션은 애플리케이션에서 캐시 기능을 
쉽게 적용할 수 있도록 제공되는 선언적(Declarative) 캐시 처리 방식입니다. 
개발자가 직접 캐시 로직을 구현하지 않아도 어노테이션만으로 캐시 조회, 저장, 삭제 등의 기능을 사용할 수 있습니다.
Spring은 캐시에 대한 추상화를 제공하며, 실제 캐시 저장소는 
다양한 구현체를 사용할 수 있습니다. 

### 대표적인 캐시 구현체

- Redis
- Caffeine

## 1. @Cacheable

@Cacheable은 메서드의 실행 결과를 캐시에 저장하고,
동일한 요청이 들어오면 메서드를 실행하지 않고 캐시된 데이터를 반환하는 어노테이션입니다.


1. 메서드 호출 시 먼저 캐시에서 데이터 조회
2. 캐시에 데이터가 존재하면 메서드를 실행하지 않고 캐시 데이터 반환
3. 캐시에 데이터가 없으면 메서드를 실행
4. 실행 결과를 캐시에 저장 후 반환


## 2. @Cacheable 사용 상황
  
  - 자주 조회하는 데이터 
  - 계산 비용이 높은 데이터

## 3. @CachePut

@CachePut은 메서드를 항상 실행하면서 실행 결과를 캐시에 갱신하는 어노테이션입니다.
@Cacheable과 달리 캐시 존재 여부와 상관없이 항상 메서드가 실행됩니다.

## 4. @CachePut 사용 상황

  - 데이터 업데이트
  - 캐시 데이터 강제 갱신

## 5. @CacheEvict

@CacheEvict는 캐시에 저장된 데이터를 제거하는 어노테이션입니다.
데이터가 삭제되거나 변경될 때 캐시를 제거하여 데이터 불일치 문제를 방지할 수 있습니다..

## 6. @CacheEvict 사용 상황

  - 캐시 데이터 삭제 혹은 변경할 때
