# AOP(Aspect Oriented Programming) 에 대해 알아보자

## 1. AOP 란?

AOP 는 Aspect Oriented Programming 의 약자로 관점 지향 프로그래밍을 의미합니다. 코드에서 반복되는 공통 기능을
관점 기준으로 정의하고 한 곳에 모아 구현 및 관리하는 프로그램 패러다임 입니다.
핵심적인 관점과 부과적인 관점으로 각각 로직을 나누어서 모듈화 하는 철학입니다.

## 2. AOP의 장점

AOP는 공통된 기능을 하는 코드를 모듈화 해 관리하기 때문에 코드 재사용성, 유지 보수성, 테스트 용이성 등의
장점을 가지고 있습니다. 특히나 핵심 비즈니스 로직과 부가 로직이 분리되어 있어 부가 로직 구현을
반복적으로 하지 않고 비즈니스 로직 구현에 시간을 많이 할애할 수 있는 것이 강력한 장점입니다.

## 3.AOP 활용 분야

- 로그 출력
- 트랙잭션 처리
- 예외 처리
- 보안/인증
- 실행 시간 체크
- 로컬 캐싱 적용

위에 나열한 항목들이 대표적으로 AOP를 활용해 미리 구현한 영역입니다. 어느 서비스에나 공통적으로 필요한
부가 로직으로, 이런 부가적인 로직은 서비스 계층마다 반복적으로 구현하기 보다는 한 곳에 모아서 관리하는
것이 효율적입니다.


## 4. AOP 동작 방식 (Spring)

1. Aspect
2. Advice
3. Join Point
4. PointCut
5. Weaving

Spring에서 AOP를 구현하는 일련의 과정입니다. 구현할 클래스를 Aspect로 정의합니다. Aspect 실행 시기를
Advice로 정의합니다. AOP 적용 지점을 Join point로 정의합니다. 어떤 메소드에 AOP를 적용할지 Point Cut으로
정의합니다. 마지막으로 핵심 비즈니스 로직에 실제로 aspect를 적용하는 과정을 Weaving이라고 합니다.


## 5. AOP에 대한 개인적 생각

AOP는 개발자가 핵심 비즈니스 로직에만 집중할 수 있게 만드는 Spring 철학에 도움을 주는 핵심적인 패러다임입니다.
AOP를 통해 부가 기능을 중복 구현하지 않을 수록 개발자의 생산성은 획기적으로 증가합니다.
