# 검증이 필요한 이유

## 1. 검증이란? 

원하는 입력 값이 올바로 들어오는 지 , 허용 가능한 데이터만 처리하고 잘못된 데이터를 차단하는지 
확인하는 과정

## 2. 계층별 검증 책임 분리

- 중복 코드 방지를 위해 계층별 검증 분리
- 책임을 명확히 함

## 3. Presentation Layer

- null 여부
- 필수 값 존재 여부
- 길이, 형식 (email, uuid 등)
- 타입 오류

비즈니스 규칙 보다는 DTO 단계에서 검증합니다. 사용자의 잘못된 요청을 빠르게
차단하는 것이 목적입니다. 

## 4. Domain Layer

- 도메인 규칙
- 불변 조건
- 상태 전이 조건

도메인 관점에서 데이터 무결성을 보장하는지 검증하고, 데이터 불변성을 검증합니다.
## 5. Application Layer

- 중복 여부
- 상태 기반 검증
- 권한 / 소유권 검증

존재해도 되는 객체인지와 객체가 가지고 있는 권한을 검증합니다.
