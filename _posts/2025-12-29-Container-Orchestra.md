# 컨테이너 오케스트레이션의 필요성

## 1. 컨테이너 오케스트레이션이란?

컨테이너 오케스트레이션은 다수의 컨테이너를 자동으로 배포,관리,확장,복구하는 시스템입니다. Docker를 단독으로 사용할 시
발생하는 관리의 다양한 난해함

 - 서버 여러대일때 배포의 복잡함
 - 설정 상태 코드 제어의 어려움
 - 트래픽 증가시 스케일링

등을 해결하기 위해 사용되는 도구입니다.

## 2. 컨테이너 오케스트레이션 역할

 컨테이너 오케스트레이션은 Docker를 단독으로 사용할 시 개발자가 직접 설정해야 하는 번거로움을 해결해줍니다. 대표적으로

 - 자동 확장(Autoscaling)
 - 자기 회복(Self-Healing)
 - 선언적 인프라

스케일링을 할 때 컨테이너 오케스트레이션은 CPU/메모리/요청 수 기반으로 자동 스케일링을 하고, 컨테이너 수를 제어해 줍니다.
또한 컨테이너의 비정상적 종료를 감지 할 경우 자동으로 새로운 컨테이너를 실행합니다. 마지막으로 오케스트레이션은 개발자가
원하는 상태를 선언시 자동으로 선언한 상태와 동일하게 컨테이너를 맞춤으로 관리합니다.

## 3. Docker vs 컨테이너 오케스트레이션

| 항목 | Docker 단독 | 컨테이너 오케스트레이션 |
|------|-------------|--------------------------|
| 컨테이너 실행 | 수동 | 자동 |
| 장애 복구 | 직접 재시작 | 자동 복구 |
| 확장 | 수동 | 자동 |
| 배포 전략 | 단순 | 롤링 / 무중단 |
| 상태 관리 | 명령 기반 | 선언적 |
| 다중 노드 관리 | 어려움 | 기본 제공 |


## 4.컨테이너 오케스트레이션 툴

- Kubernetes
- Docker Swarm
- Nomad

이 중 Kubernetes가 가장 대중화되고 표준으로 사용되고 있습니다.

## 5. 개인적 생각

소규모 프로젝트에서는 Docker를 일일히 관리할 수 있지만, 대규모 프로젝트에서는 Docker의
자동화는 필수 항목이라고 생각합니다.
