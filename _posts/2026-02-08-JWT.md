# JWT(JSON Web Token)의 구조와 각 구성 요소


## 1. JWT 란?

JWT(JSON Web Token)는 클라이언트와 서버 간에 인증 및 인가 정보를 안전하게 전달하기 위한 
토큰 기반 인증 방식입니다.
서버가 사용자 정보를 토큰에 담아 서명(Signature)한 뒤 클라이언트에 전달하고
, 이후 요청마다 해당 토큰을 검증함으로써 사용자를 식별합니다.

## 2. JWT 구조

  - Header 
  - Payload
  - Signature

## 3. JWT 구성 요소

  - Header : 토큰 타입과 서명 알고리즘 정보 
  - Payload: 사용자 정보 및 토큰 클레임
  - Signature: 토큰 검증을 위한 값

## 4. JWT 장점 

1) Stateless 인증
서버가 세션을 저장하지 않아도 되서
수평 확장(Scale-out)에 유리합니다.
2) 분산 시스템에 적합
여러 서버에서 동일한 토큰으로 인증 가능 합니다

3) 빠른 인증 처리
DB 조회 없이 토큰 검증만으로 인증 가능합니다.
4) 확장성
Payload에 필요한 정보를 자유롭게 추가 가능합니다.
