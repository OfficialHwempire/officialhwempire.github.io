# DB 정규화와 역정규화

## 1. 정규화란?

정규화 (Normalization)은 데이터 베이스 설계 과정에서 데이터 중복을 방지하고, 데이터 구조를
안정화 하기 위해 DB 테이블을 체계적으로 분리하는 과정입니다. 제 1,2,3 정규화 등을 통해
중복 제거 및 데이터 정합성을 향상할 수 있습니다.

## 2. 역정규화란?

역정규화(Denormalization) 은 정규화와 반대로 데이터를 의도적으로 중복하는 기법입니다.
정규화 과정에서 데이터 테이블을 지나치게 Join 하면서 발생하는 성능 하락을 개선할려는 목적으로
사용합니다.

## 3. 역정규화의 장점

  - Join 으로 인한 조회 성능 저하 방지
  - 실시간 응답 속도 향상

정규화는 데이터를 조회할 때 필요한 행동량을 필연적으로 늘립니다. 역정규화는 데이터 중복을 통해,
불필요한 Join 과정을 없애 조회 및 DB 응답 속도를 확연하게 증가시킵니다.

## 4. 역정규화 예시

 - 통계 시스템 집계 테이블 생성 서비스
 - 게시판 댓글 수 게시글 테이블에 중복 저장

기본적으로 반복적인 조회를 제공해야 하는 서비스에서 성능 향상을 위해 역정규화를 사용합니다.
DB의 데이터가 많을 수록 정규화를 통한 집계 함수 및 통계 함수의 성능이 현저히 저하합니다. 이를 방지하기 위해서
역정규화를 사용합니다. 대략 수백배 이상의 조회 속도 차이를 낼 수 있습니다. 

## 5. 역정규화의 단점

 - 데이터 중복 증가
 - 데이터 정합성 유지 어려움
 - 데이터 수정 및 삭제 시 비용 증가
 - DB 테이블 구조 복잡도 증가로 인한 유지 보수 비응 증가

조회 성능이 개선되지만 그 이외의 영역에서 유지 보수 비용 및 데이터
삽입,삭제 , 변경시 발생하는 비용이 증가합니다.

## 6. 역정규화에 대한 개인적 생각

자주 호출되는 조회 서비스에는 필수적으로 역정규화를 적용해야 합니다.
정규화와 수백배에서 수천배의 속도 차이가 납니다. 자주 호출하는
조회 서비스의 응답 처리 속도는 서비스 사용 경험과 직접적으로 연관이
있습니다. 역정규화는 서비스를 제공할 때 필연적으로 수행해야 하는 사항입니다.
