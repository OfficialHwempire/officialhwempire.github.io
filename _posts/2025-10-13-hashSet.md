# 0. [JAVA] Set 이란?

오늘 알아볼 HashSet 은 Set 인터페이스를 구현한 클래스입니다. Set 인터페이스는 2가지 특징을 가지고 있습니다.

1. Key 의 중복을 허용하지 않는다
2. 비선형 구조로 원소의 순서 및 인덱스가 존재하지 않는다. 

Set 자료형은 Key 중복을 자동으로 제거해 줍니다. 인덱스가 존재하지 않아 내부 탐색 및 값을 변경할 때 List 보다
속도가 느립니다.


# 1. HashSet 이란?

Set 인터페이스는 기본적으로 <Key, Value> 형태로 저장합니다. HashSet은 Set 인터페이스를 구현한 클래스입니다.
Set의 Key 값이 중복되지 않는 특성을 이용해 HashSet 내부엔
중복된 객체가 존재하지 않습니다.

# 2. HashSet 선언

```java
HashSet<Integer> set = new HashSet<>();
set.add(1);
set.add(2);
```

Java 에서 HashSet 은 HashSet<T> 변수명 = new HashSet<T>(); 형식으로 선언합니다.

# 3. HashSet 작동 방식

HashSet은 HashMap의 <Key, Value> 형태에서 저장할 객체를
Key 로 사용하고 Value 에는 값이 고정된 더미 객체를 사용합니다.
HashSet에 새로운 객체를 추가할 때 HashSet은 새로운 객체의 hashCode() 메소드를 호출합니다. 이 메소드는
객체의 메모리 주소 등을 기반으로 고유한 정수값을 반환합니다. 이렇게 생성된 해시코드는 HashSet의 내부 저장 위치(Bucket)를
결정합니다. 만약 동일한 해시 값을 가진 객체가 이미 HashSet에 존재한다면 새로운 객체를 저장하지 않습니다.

# 4. HashSet 특징

1. <b>null </b> 값을 허용한다
2. HashTable 구조를 따른다
3. 요소의 순서가 존재하지 않느다.

# 5. HashSet 활용도 

1. <b>중복 제거</b> : 자동으로 중복된 객체를 제거합니다
2. 향상된 데이터 처리 기능: 요소 탐색 , 추가, 삭제에 <b>O(1)</b>  시간복잡도를 가지고 있어 빠른 데이터 처리가 가능합니다
