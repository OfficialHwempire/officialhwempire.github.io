# 세션 기반 인증 VS 토큰 기반 인증


## 1. 세션 기반 인증이란?

세션 기반 인증은 사용자가 애플리케이션에 접속했을 때 혹은 로그인 하였을시 사용자 정보를 세션으로 저장하는 방식입니다.
서버는 세션을 저장하고, 클라이언트에 세션 정보가 담긴 쿠기를 전송합니다. 클라이언트는 쿠키를 통해 서버로 부터
인증을 받을 수 있습니다. 서버 메모리 혹은 REDIS 를 사용하여 상태를 유지하는 Stateful 방식입니다.

## 2. 세션 기반 인증 보안 고려사항

 - 세션 탈취 방지를 위해 HTTPS 필수
 - 쿠키 취약점 방지를 위해 HTTPONLY, SECURE,SAMESITE 등의 설정 필요
 - CSRF 공격 방지를 위한 CSRF 토큰 필요
 - 서버 마이그레이션 시 세션 동기화 필요

## 3. 토큰 기반 인증이란?

사용자 인증 후 클라이언트가 서버에 요청을 보낼시 토큰을 지급합니다. 토큰을 통해 클라이언트를 인증합니다.
서버가 사용자 인증 상태를 저장하지 않는 Stateless 방식입니다.

## 4. 토큰 기반 인증 보안 고려사항

  - 토큰 악용 방지를 위해 토큰 만료시간 설정 필요
  - HTTPS 필수
  - XSS 방지를 위해 토큰 저장 위치 관리 필요
  - 보안을 위해 REFRESH 토큰 관리 필요

## 5. 세션 기반 인증 VS 토큰 기반 인증

세션 기반 인증과 토큰 기반 인증의 가장 큰 차이점은 서버가 인증 상태를 저장하는지 여부입니다.
기본적으로 보안이 많이 요구되는 분야는 서버가 모든 정보를 제어하고 관리하는 세션 기반 인증 방식을
채택합니다. 정보 처리 , 통신 비용이 부담되는 모바일 , 웹 환경에는 Stateless 토큰 방식이 유지 보수 및
확장, 성능 면에서 유리합니다.
