# DDL VS DML


## 1. DML 이란 ?

DML (Data Manipulation Language) 는 DB 테이블의 실제 데이터를 제어하는
명령어 입니다. CRUD 에 해당하는 조회, 삭제,수정, 삽입을 진행할 수 있습니다.
SELECt, INSERT, UPDATE, DELETE 가 DML 명령에 해당합니다.

## 2. DDL  이란 ?

DDL(Data Definition Language) 은 DB 테이블의 전체 구조를 변경하는 명령어입니다.
테이블 내부 데이터가 아닌 테이블 전체를 생성,변경, 삭제할 수 있습니다.
CREATE,ALTER,DROP 이 DDL 명령어에 해당 합니다.

## 3. DDL 종류


| 명령어          | 설명                          |
| ------------ | --------------------------- |
| **CREATE**   | 테이블·뷰·인덱스 등 **새 객체 생성**     |
| **ALTER**    | 테이블 구조 **변경** (컬럼 추가/수정/삭제) |
| **DROP**     | 테이블·뷰·인덱스 등 **객체 삭제**       |
| **TRUNCATE** | 테이블 **데이터 전체 삭제**, 구조는 유지   |
| **RENAME**   | 객체 이름 변경                    |


## 4. DML 종류

| 명령어        | 설명        |
| ---------- | --------- |
| **SELECT** | 데이터 조회    |
| **INSERT** | 데이터 삽입    |
| **UPDATE** | 기존 데이터 수정 |
| **DELETE** | 데이터 삭제    |

## 5. DDL vs DML

DDL과 DML은 명령어가 작용하는 대상이 다릅니다. DDL은 DB 테이블 자체를 제어하는 역할을 합니다.
DML은 테이블 내부의 데이터를 제어하는 역할을 담당합니다. 행위의 저장 단계도 다릅니다. DDL은 
명령 실행시 자동으로 COMMIT 이 진행됩니다. DML은 트랜잭션을 적용할 수 있어 이미 실행한 DML 명령어도
ROLLBACK 할 수 있습니다.
